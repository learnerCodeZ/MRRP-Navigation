


# UI 视觉风格与实现规范（用于喂给 Claude）

请根据以下视觉风格规范，为我的 Unity (MRTK) 项目编写 UI 界面样式、材质属性、以及动态布局代码：

## 1. 整体视觉风格 (Style Overview)

* **核心风格**：工业感、科幻、轻量、半透明悬浮面板（Diegetic Glassmorphism）。
* **设计原则**：信息密度低、对比度高。不采用大面积死板的纯色，而是使用带有细腻渐变的半透明材质，让用户能透过 UI 看到真实世界的物理地面。

## 2. 颜色系统 (Color Palette)

* **主色调（面板背景）**：深蓝色/藏青色偏紫。
* *Hex 颜色参考*：`#1A255C` 到 `#2C3A8A`
* *不透明度 (Alpha)*：保持在 `60% - 80%` 之间，具有明显的玻璃通透感。


* **强调色/前景色（文字与图标）**：纯白色 (`#FFFFFF`)。
* **轨迹与标记色（HRP 路径）**：
* *未激活/基础状态*：亮蓝色 (`#4DEEEA`)。
* *画线球体（小蓝球）*：青蓝色半透明发光体。
* *特殊标记（如终点/目标）*：明黄色 (`#FFD700`) 的立体几何体（如正八面体/菱形）。



## 3. 排版与字体 (Typography)

* **字体类型**：无衬线体（Sans-Serif），如 Arial, Segoe UI 或 Inter。严禁使用带修饰的复杂字体。
* **字重与大小对比**：
* *大标题/状态栏（如 STAGE 0 / ADD MODE）*：加粗（Bold），字号较大，全大写。
* *按钮文字*：常规（Regular）或中粗（Medium），字号适中，居中对齐。


* **排版特征**：文字和图标呈垂直堆叠布局，字距和行距拉开，营造出开阔、严谨的工业仪表感。

## 4. 界面几何与布局结构 (Geometry & Layout)

* **横幅状态栏 (Top Banner)**：
* *位置*：悬浮在场景正上方或面板顶部。
* *形状*：长方形，无圆角（硬直角边缘），极简边缘线。
* *层级*：分为两行，第一行为辅助小字（如 Stage 0），第二行为核心大字（如 OFF MODE / ADD MODE）。


* **操作主面板 (Main/Preferred Menu)**：
* *结构*：采用 `2x2` 网格布局或 `1x3` 垂直长条布局。
* *间距*：按钮之间留有大面积的留白，避免视觉拥挤。
* *边缘*：直角边框，没有现代手机 UI 常见的大圆角，强化工业和学术严谨感。



## 5. 图标与按钮设计 (Icons & Buttons)

* **图标风格**：极简线条风（Line Icon / Wireframe），单色纯白，线条纤细，无拟真和阴影。
* *ADD*：纤细的 `+` 号。
* *CLEAR*：放射状的线条或错号 `X`。
* *SEND*：线条卡车（🚚 符号的极简线框版）。
* *Back*：标准的纤细左箭头 `←`。


* **交互动效**：当射线悬停（Hover）或手指接近时，按钮背景的 Alpha 值或亮度应有轻微的提亮反馈。
